# 贡献指南

欢迎补充课程攻略、原创练习、纠错建议和已经确认可以公开的学习资料。

## 投稿流程

1. 先搜索现有课程目录和 Issue，避免重复；
2. Fork 仓库并创建只包含一门课程或一类资料的分支；
3. 把资料放入对应课程的 `攻略/`、`考试/` 等目录，并更新课程 README；
4. 运行 `python tools/check_repo.py .` 和 `python -m unittest discover -s tests -v`；
5. 提交 Pull Request，如实填写来源、许可、隐私检查和考试状态。

不熟悉 Git 时，可以先使用“资源投稿建议”Issue。不要直接在 Issue 中上传未经审核的文件、授权聊天记录或个人信息。

## 文件与目录

- 每门课程使用一个根目录，并必须包含 `README.md`；
- 文件名使用简短、可辨认的中文或英文，不得包含姓名、学号、班级、手机号等信息；
- 单个文件必须小于 25 MiB；
- 不提交 ZIP/RAR、安装程序、危险宏、依赖目录、缓存和构建产物；
- 完整软件项目应清理账号、密钥与构建物后放入独立源码仓库，本仓库只保留说明和链接。

## 试卷与答案

- 真实试卷只能在对应考核已经结束、取得方式正当且具有公开依据后提交；
- 禁止上传正在进行或即将使用的试卷、未公开题库、命题文件和保密材料；
- 模拟卷必须明确标注，不得冒充历年原卷；
- 学生整理答案必须显示“非标准答案，仅供复习核对”，并说明核验程度；
- 考试结束并不等于自动取得版权许可。

## 权利与隐私确认

提交者必须确认内容为本人原创、采用明确开放许可，或已经取得允许公开转载的授权。公开前还应检查正文、批注、修订记录、Office/PDF 属性和图片 EXIF，移除不必要的个人信息与凭据。

授权证明含个人信息时不要提交到公开仓库，只在维护者需要核验时通过私密方式提供最少信息。

## 许可

提交本人原创的攻略、笔记、模拟题或解答，即表示同意按 CC BY-NC-SA 4.0 公开。第三方内容继续适用其原许可证或授权条件。
